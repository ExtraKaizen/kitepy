## Overview

kitepy provides built-in profiling tools to understand your model's performance characteristics: parameter count, memory usage, latency, and throughput.

## Quick Profile

Get a performance summary with one call:

```python
from kitepy import CNN

model = CNN("resnet50", num_classes=10)
profile = model.profile()

print(f"Parameters: {profile['total_params']:,}")
print(f"Model size: {profile['model_size_mb']:.1f} MB")
print(f"Latency: {profile['latency_ms']:.1f} ms")
print(f"Throughput: {profile['throughput']:.1f} samples/sec")
```

Output:
```
Parameters: 25,557,032
Model size: 97.5 MB
Latency: 12.3 ms
Throughput: 81.3 samples/sec
```

## Profile Details

The `profile()` method returns a dictionary:

```python
{
    'total_params': 25557032,
    'trainable_params': 25557032,

    'throughput': 81.3,
    'device': 'cuda:0',
}
```

## Comparing Models

Compare different architectures:

```python
from kitepy import CNN, Transformer

models = ["resnet18", "resnet50", "efficientnet_b0", "vit_tiny", "vit_base"]

print(f"{'Model':<20} {'Params':>12} {'Size (MB)':>10} {'Latency':>10}")
print("-" * 55)

for name in models:
    if name.startswith("vit"):
        model = Transformer(name, num_classes=10)
    else:
        model = CNN(name, num_classes=10)
    
    profile = model.profile()
    print(f"{name:<20} {profile['total_params']:>12,} {profile['model_size_mb']:>10.1f} {profile['latency_ms']:>10.1f}ms")
```

Output:
```
Model                     Params  Size (MB)    Latency
-------------------------------------------------------
resnet18             11,689,512       44.6       5.2ms
resnet50             25,557,032       97.5      12.3ms
efficientnet_b0       5,288,548       20.2       8.1ms
vit_tiny              5,717,512       21.8       4.1ms
vit_base             86,567,656      330.2      23.5ms
```

## Latency Benchmarking

### Single Image Latency

```python
profile = model.profile(batch_size=1)
print(f"Single image latency: {profile['latency_ms']:.2f} ms")
```

### Batch Latency

```python
profile = model.profile(batch_size=32)
print(f"Batch latency: {profile['latency_ms']:.2f} ms")
print(f"Per-image: {profile['latency_ms'] / 32:.2f} ms")
```

### Warmup

Run warmup iterations before measuring:

```python
profile = model.profile(warmup_runs=10, timed_runs=100)
```

## Optimization Recommendations

After profiling, try these optimizations:

### Reduce Model Size

```python
# Use a smaller architecture
CNN("resnet18")  # Instead of resnet50

# Or EfficientNet for better accuracy/size tradeoff
CNN("efficientnet_b0")
```

### Improve Latency

```python
# Use quantization
model.quantize(bits=8)

# Use mixed precision inference
model.half()  # FP16
```

### Improve Throughput

```python
# Use larger batch sizes
profile = model.profile(batch_size=64)

# Enable cudnn.benchmark
import torch
torch.backends.cudnn.benchmark = True
```

## Profiling During Training

Track metrics during training:

```python
from kitepy.core.callbacks import ProfilerCallback

profiler = ProfilerCallback(log_every_n_steps=100)
model.train("cifar10", callbacks=[profiler])

# Get stats
print(profiler.get_stats())
```

## When to Profile

Profile your models:

- **Before training**: Choose appropriate architecture
- **After training**: Before deployment
- **During optimization**: Compare quantized/pruned versions
- **For deployment**: Ensure latency requirements are met
