## Overview

Use kitepy's training infrastructure with your custom PyTorch model.

## Full Code

```python
"""
Using a custom PyTorch model with kitepy.
"""
import torch.nn as nn
from kitepy import CNN

# Define custom architecture
class CustomCNN(nn.Module):
    def __init__(self, num_classes=10):
        super().__init__()
        self.features = nn.Sequential(
            nn.Conv2d(3, 64, 3, padding=1),
            nn.BatchNorm2d(64),
            nn.ReLU(),
            nn.MaxPool2d(2),
            
            nn.Conv2d(64, 128, 3, padding=1),
            nn.BatchNorm2d(128),
            nn.ReLU(),
            nn.MaxPool2d(2),
            
            nn.Conv2d(128, 256, 3, padding=1),
            nn.BatchNorm2d(256),
            nn.ReLU(),
            nn.AdaptiveAvgPool2d(1),
        )
        self.classifier = nn.Linear(256, num_classes)
    
    def forward(self, x):
        x = self.features(x)
        x = x.flatten(1)
        x = self.classifier(x)
        return x

def main():
    # Create custom model
    custom_model = CustomCNN(num_classes=10)
    
    # Wrap with kitepy
    model = CNN(custom_model)
    
    # Train using kitepy's infrastructure
    model.train(
        "cifar10",
        epochs=20,
        lr=0.001,
        batch_size=64,
    )
    
    # All kitepy features work!
    model.save("custom_model.pt")
    metrics = model.evaluate("cifar10")
    print(f"Accuracy: {metrics['accuracy']:.2f}%")

if __name__ == "__main__":
    main()
```

## Benefits

When you pass a custom model to kitepy, you get:

- ✅ Automatic device handling
- ✅ Mixed precision training
- ✅ Learning rate scheduling
- ✅ Checkpointing
- ✅ Logging and callbacks
- ✅ Save/load functionality

## nn.Sequential Shorthand

```python
model = nn.Sequential(
    nn.Conv2d(3, 32, 3, padding=1),
    nn.ReLU(),
    nn.MaxPool2d(2),
    nn.Conv2d(32, 64, 3, padding=1),
    nn.ReLU(),
    nn.AdaptiveAvgPool2d(1),
    nn.Flatten(),
    nn.Linear(64, 10)
)

CNN(model).train("cifar10")
```
